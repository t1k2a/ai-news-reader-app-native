# フェーズ1.5 マニュアルプレイブック
## 目標：最初の10〜30フォロワーを獲得する

**期間目安:** 2〜4週間
**移行条件:** フォロワー10人以上 / 平均インプレッション50+/投稿 / サイトへの1件以上のクリック

---

## なぜ手動施策が必要か

自動投稿はすでに稼働中だが、フォロワー0の状態では誰のタイムラインにも表示されない。
X のアルゴリズムは**エンゲージメント（いいね・リプライ）が先にあって初めて拡散を始める**。
コード改善だけでは解決できないため、最初の10〜30人は手動アクションで獲得する。

---

## 日次ルーティン（30分/日）

### 朝（10分） — 投稿確認
1. 自動投稿の最新ツイートを確認（GitHub Actions は JST 7:00 / 12:00 / 20:00 に実行）
2. 投稿が正しく表示されているか確認（画像あり・スレッド形式）
3. いいねが来ていたら必ずお礼リプライ

### 昼・夜（20分） — エンゲージメント

**検索クエリ（X の検索バーに入力）:**
```
#AIニュース lang:ja
#生成AI lang:ja
ChatGPT 使ってみた lang:ja
Claude 試した lang:ja
LLM 日本語
```

**アクション手順:**
1. 上記クエリで検索し、直近1時間以内の投稿を表示
2. **有益・具体的なリプライを3〜5件** 投稿する
   ❌ 「参考になります！」だけはNG
   ✅ 追加情報・自分の実体験・関連記事の紹介
3. 気に入ったツイートに**いいね** も押す（リプライの13.5倍コストで効果あり）

## フォロー戦略（初期50件リスト）

以下のカテゴリで日本語AIアカウントを計50件フォローする。
フォローされたら高確率でフォロバしてもらえるアカウントタイプを優先する。

### カテゴリA: AI情報発信アカウント（15件）
X 検索で `#AI情報 OR #AIニュース` + フォロワー1,000〜10,000 のアカウントを探す
（大手ではなくニッチアカウントが互いにフォローし合う傾向が強い）

### カテゴリB: AI活用・実践系アカウント（15件）
検索: `ChatGPT 活用事例 lang:ja` のツイート主
特に「毎日AIを使っている」「プロンプト集を作っている」系のアカウント

### カテゴリC: テック系個人ブロガー・エンジニア（10件）
Zenn / Qiita でAI記事を書いている人がXアカウントを持っている場合が多い

### カテゴリD: AIメディア・コミュニティ（10件）
- AI関連のSlack・Discord コミュニティの管理者
- 「AI Weekly Japan」「AIタイムズ」等の非公式まとめ系

**注意:** 1日のフォロー上限は400件（Xのポリシー）。1日10〜20件程度に抑えて自然に見せる。

## コンテンツミックス戦略

### 自動投稿（毎日3回、既存）
- RSSフィードから最新ニュースを翻訳投稿
- 画像付き・スレッド形式で可視性を向上

### 手動投稿追加（週1〜2回）

#### パターン1: 週次AIまとめスレッド（月曜日推奨）
```
【今週のAIニュース TOP5】 🔥

1/ OpenAIが〇〇を発表。...（要約1〜2行）

2/ Googleが〇〇を〇〇に...

...

5/ ...

詳細はこちら👇 https://glotnexus.jp

#AIニュース #海外AI #生成AI
```

#### パターン2: 「〇〇について調べた」系ツイート
- 最近注目の技術トピックについて自分の考えを短くまとめる
- 例: 「GPT-4oの音声機能について調べた。日本語対応の精度が思ったより高かった件」

#### パターン3: Xite/引用RT
- 海外AIニュースを引用RT して日本語で一言コメントを追加
- 例: `"これ日本語でまとめると〇〇ということ。実務への影響は..."`

## ハッシュタグ戦略（改訂版）

エージェント調査で「#GlotNexus と #海外のAIニュース は検索ボリューム0」と判明。
以下に修正することを推奨する。

### 自動投稿のハッシュタグ（現在）
```
#AI #海外のAIニュース #GlotNexus
```

### 推奨タグ（検索ボリュームあり）
```
#AI #生成AI
```
**2タグのみ** に絞ることで、各タグのリーチ計算でペナルティを受けにくくなる。
（Xのアルゴリズムはタグ多用を spam と判断する傾向がある）

> ⚠️ タグの変更は `lib/auto-post.ts` と `lib/auto-post-enhanced.ts` を修正する必要がある。
> 変更前にA/Bテストで効果を比較することを推奨。
